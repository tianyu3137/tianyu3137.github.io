title: 个人博客建立第二步-配置git
date: 2016-05-12 09:07:35
tags: hexo Github node.js

安装完hexo，拥有了自己的blog文件夹之后，我们就可以进入搭建个人博客的最后一步，配置git。

配置Github-建立Repository

建立与你用户名对应的仓库，仓库名必须为【your_user_name.github.io】,此为固定写法，不可更改。

建立新仓库的步骤，登入个人github账号，点击下图中的New Repository

然后会进入如下页面
在Repository name处填写仓库名，格式必须是【your_user_name.github.io】！！！（如图，我的Repository name即tianyu3137.github.io
参照这格式填写）可以在Description处添加描述，选择Public，勾上README，最后点击Create repository即可！

关联仓库与本地文件夹

进入你建立的blog文件夹，以我这为示(版本不同文件夹可能略有变化，但没有影响),使用编辑器进入_config.yml文件，我们需要通过它来建立关联，翻到最下面，将deploy下的type和repo改成图中这样的，记得将repo下的tianyu3137改成你自己的用户名！

PS:在这之前需要绑定SSH Keys，如果你不会可以搜索网上教程，在此不作叙述。

然后在终端你的blog文件夹下运行命令: npm install hexo-deployer-git —save.

成功后继续运行命令，hexo g，成功后继续运行hexo d即可！

之后在浏览器输入xxx.github.io（把xxx改成你的用户名），就能看到属于自己的博客了！

每次更新博客后，只需重新运行hexo g和hexo d 就可以同步更新个人博客！

常用命令

hexo new “postName” 新建文章

hexo new page “pageName” 新建页面

hexo g 生成静态页面至public目录

hexo server 开启预览访问端口（默认端口4000，ctrl+c关闭server）

hexo d 将.deploy目录部署到GitHub

hexo version 查看Hexo的版本

git clone 下载线上仓库地址

git status 查看当前的仓库状态

git add .

git commit -m ‘修改的内容提要’ 提交到缓存区

个人博客建立第一步之hexo

今天学习了使用HEXO+Github搭建一个属于自己的博客，方法简单，耗时较少，现在将搭建方法分享给大家~（以下教程以mac系统为例）

配置环境

安装Node.js（必装）

作用：用于生成静态页面
可去Node.js官网下载对应的最新版本，一直点下一步直至完成即可。

安装Git（必装）

作用：把本地的hexo内容提交到github上
只需安装Xcode即可，里面自带Git

申请Git账号（必须）

作用：用来做博客的远程创库、域名、服务器之类
申请git账号可去git官网https://github.com/进行申请注册，流程与一般的注册账号差不多。（如何配置SSH KEYS会在下一节讲到）

安装Hexo

在这之前，先要了解终端（即windos下的命令提示符）的常用命令与使用方法

终端位于LaunchPad的其他下，常用命令如下（注意：所有命令后都需加上一个空格）
ls: 显示当前目录
cd 某名称: 进入某目录 例子： cd Desktop/Users/lanou/HBuilderProjects/个人博客-blog/source/_posts/个人博客建立之hexo2.md: 进入桌面

cd ..: 返回上层目录 cd ../../../ 返回上几层

mKdir 文件夹名称: 创建文件夹

touch 文件名称: 创建文件

rm 文件名称: 删除文件

rm -rf 文件夹名称: 删除文件夹

pwd+空格:查看当前文件的目录路径

ifconfig:查看本地的

在了解了上面的终端命令操作后，建议在桌面新建一个bolg文件夹作为本地存储博客的文件夹，然后通过终端命令进入该文件夹里安装Hexo。

执行如下命令安装Hexo: sudo npm install -g hexo（-g代表全局，即安装在系统里的意思）

安装之后，执行init命令初始化hexo，命令:

hexo init

至此，全部安装工作已经完成！blog就是你的博客根目录，所有的操作都在里面进行。

最后，hexo generate (简写hexo g) ,生成静态页面。

hexo server 在浏览器输入’http://localhost:4000'，预览本地页面
